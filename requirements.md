# 页面复刻需求文档

## 1. 目标

完整复刻竞彩足球数据站（以 `cp.titan007.com` 为例）的比赛列表页及其关联页面，保证复刻后的页面在浏览器中可直接打开、保留原始样式与交互按钮，并能逐层下钻浏览。

## 2. 复刻层级定义

以单场比赛为线索，将整个站点划分为三个层级。每一层页面都需要完整保留 CSS、图片、按钮和可见交互元素；进入下一层的入口链接需全部改写成复刻后的本地路径。每一层内部的所有标签页切换、勾选、筛选操作所产生的可见内容，都属于该层级的复刻范围。

### 2.1 第一层级（L1）——比赛列表页

- **入口**：带日期的列表页，例如
  `https://cp.titan007.com/buy/JingCai.aspx?typeID=101&oddstype=2&date=2026-7-21`
- **内容**：当天全部比赛的列表，包含联赛、开赛时间、主队、客队、比分、赔率、操作按钮等。
- **可进入 L2 的入口**（以下入口点击后均进入 L2）：
  - 点击比赛行中的 **赛事类型名称**（如“欧冠杯”），进入该赛事类型/联赛的聚合页或资料页。
  - 点击比赛行中的 **亚**、**欧**、**析** 按钮，分别进入单场亚盘详情页、单场欧赔详情页、单场分析页。
- **保存位置**：`output/{date}/index.html`

### 2.2 第二层级（L2）——聚合页与单场详情页

L2 包含两类页面，它们处于同一层级：

- **类型 A：赛事类型/联赛聚合页**
  - **入口**：在 L1 点击比赛行中的 **赛事类型名称** 进入。
  - **内容**：该赛事类型/联赛下的比赛集合、联赛资料、排名、近期战绩等。
  - **交互**：该页内部的所有 **标签页切换**、**勾选框**、**筛选条件** 等产生的可见内容都属于 L2 复刻范围，需要完整保留。
  - **进入 L3 的入口**：在该聚合页中点击每场比赛的 **亚**、**欧**、**析**、**大**，进入对应单场比赛深度页（L3），共 4 个页面。

- **类型 B：单场详情页**
  - **入口**：在 L1 点击单场比赛的 **亚**、**欧**、**析** 进入，对应 URL 为：
    - **亚**：单场亚盘详情页 `//vip.titan007.com/AsianOdds_n.aspx?id={match_id}`
    - **欧**：单场欧赔详情页 `//op1.titan007.com/oddslist/{match_id}.htm`
    - **析**：单场分析页 `//zq.titan007.com/analysis/{match_id}cn.htm`
  - **内容要求**：
    - 页面首屏及由标签页、勾选、筛选产生的所有可见内容必须完整复刻。
    - 页面内部的 **标签页切换**、**勾选框**、**筛选条件** 等交互组件属于 L2 范围，需要一并复刻。若这些交互会切换 DOM，则每个状态都需要单独保存为本地页面，确保静态浏览时不丢失内容。
  - **进入 L3 的入口**：在单场分析页（析）中点击 **主队名称** 或 **客队名称**，分别进入主队资料库页与客队资料库页（L3）。

- **保存位置**：
  - 单场详情页：`output/{date}/{type}/{match_id}.html`
    - `{type}` 可取 `asian`、`europe`、`analysis`
  - 赛事类型/资料聚合页：`output/{date}/league/{league_id}.html`

### 2.3 第三层级（L3）——深度单场页与球队资料页

L3 包含两类入口，以单场比赛为线索共 6 个页面：

- **入口 1**：在 L2 的赛事类型/联赛聚合页中，点击每场比赛的 **亚**、**欧**、**析**、**大**，分别进入 4 个单场深度页。
  - **亚**：单场亚盘详情页 `//vip.titan007.com/AsianOdds_n.aspx?id={match_id}`
  - **欧**：单场欧赔详情页 `//op1.titan007.com/oddslist/{match_id}.htm`
  - **析**：单场分析页 `//zq.titan007.com/analysis/{match_id}cn.htm`
  - **大**：单场大小球详情页 `//vip.titan007.com/OverDown_n.aspx?id={match_id}`
- **入口 2**：在 L2 的单场分析页（从 L1 点击 **析** 进入）中，点击 **主队名称** 或 **客队名称**，分别进入 **主队资料库页** 与 **客队资料库页**。
- **内容**：单场比赛的详细数据、赔率走势、历史交锋、球队资料、联赛排名等。
- **保存位置**：
  - 球队资料库页：`output/{date}/team/{team_id}.html`
  - 其他 L3 深度页按 URL 模式映射到 `output/{date}/...` 下对应目录

## 3. 通用要求

- **渲染方式**：先使用 Headless Chromium 执行原始页面的 JS，获取渲染后的 DOM，再对 DOM 进行静态化处理。
- **静态化处理**：
  - 删除会重新请求远端数据或重新初始化界面的脚本。
  - 保留 `openAnalysisPage` 等必要的本地跳转函数，使其指向复刻后的本地页面。
  - 内联 CSS、图片等静态资源，避免跨域或混合内容问题。
- **超链接跳转关系复刻**：
  - 页面中所有指向远端同域的相对/绝对链接，只要目标页面已被复刻，统一改写成复刻后的本地相对路径；外部广告、登录等无关链接可保留原样或移除。
  - L1 列表页中每场比赛的 **赛事类型名称**、**亚**、**欧**、**析** 入口，必须链接到本地复刻的 L2 页面。
  - L2 赛事类型/联赛聚合页中每场比赛的 **亚**、**欧**、**析**、**大** 入口，必须链接到本地复刻的 L3 页面。
  - L2 单场分析页（析）中的 **主队名称**、**客队名称**，必须链接到本地复刻的球队资料库 L3 页面。
  - L2/L3 各详情页内部的返回、切换、下钻链接，若目标已被复刻，也应改写为本地路径，确保用户在复刻站点内点击任意有效入口都能连续浏览，不会跳转回源站。
- **视觉对比**：作为可选的质量检查手段；当差异过大时输出警告，但不阻塞后续层级抓取。
- **输出**：
  - 每个日期生成独立目录。
  - `meta.json` 记录比赛元数据与各级页面映射。
  - `dates.json` 汇总所有已复刻日期，供前端导航使用。

## 4. 前端浏览

- 提供一个静态首页 `docs/index.html`：
  - 顶部为日期选择器和操作按钮。
  - 日期选择后，在页面上方展示该日期的比赛列表，每场比赛显示：联赛、开赛时间、主队、客队、状态。
  - 每场比赛行额外展示复刻统计字段：
    - **L1 页应复刻面数 / 已视觉对比页面数**：整站 L1 为 1 个列表页；已视觉对比页面数按实际完成对比的 L1 页面计数。
    - **L2 页应复刻面数 / 已视觉对比页面数**：对单场比赛为 **赛事类型名称页**、**亚**、**欧**、**析** 共 4 个页面；已视觉对比页面数按实际完成对比的 L2 页面计数。
    - **L3 页应复刻面数 / 已视觉对比页面数**：对单场比赛为从赛事类型名称页进入的 **亚**、**欧**、**析**、**大** 共 4 个页面，加上从析页面进入的 **主队资料库页**、**客队资料库页** 共 2 个页面，合计 6 个页面；已视觉对比页面数按实际完成对比的 L3 页面计数。
  - 下方用 iframe 展示所选日期的 L1 列表页。
  - L1 页面中的入口点击后，在 iframe 内打开本地 L2/L3 页面。
- 页面按 PC 端宽度渲染，移动端可横向滚动查看完整内容。

## 5. 非功能要求

- 复刻结果托管在 GitHub Pages，地址为 `https://shangdabaitu.github.io/page-replica-v2/`。
- 新增日期复刻时，后端服务在本地执行，完成后自动推送静态结果到 `docs/` 并部署。
